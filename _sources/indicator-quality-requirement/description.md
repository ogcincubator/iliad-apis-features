# SeaDOTs Indicator Quality Requirement

This draft building block records what data must be true before an indicator can be used in an assessment.

It is designed to give a clear path:

`assessment model -> indicator -> quality requirement -> accepted input products`

The profile can express:

- required geometry and CRS
- minimum spatial and temporal support
- baseline/control requirements
- method metadata
- uncertainty requirements
- controlled-vocabulary requirements
- access/readiness status

It can be used with ODD `qualityRequirements` and OIM observed-property URIs.

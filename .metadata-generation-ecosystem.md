# STAC and DCAT Metadata Generation Ecosystem

## Overview

This system provides automated STAC (SpatioTemporal Asset Catalog) and DCAT (Data Catalog Vocabulary) metadata generation from data sources, with intelligent routing to specialized agents based on data format and user intent.

**Key Features:**
- 🚀 Automatic format detection from URLs or local files
- 🤖 Intelligent metadata extraction (spatial, temporal, properties)
- 🎯 Interactive field completion for missing required metadata
- 📊 Multi-format output (JSON, JSON-LD, Turtle, RDF/XML)
- 🔗 Standards-compliant with OGC geodcat-ogcapi-records
- 🔄 Batch processing for multiple data sources
- 📦 Extensible architecture for new data formats

## Data Format Support

| Format | Input | Extraction | STAC | DCAT | Specialized Agent |
|--------|-------|-----------|------|------|---|
| **NetCDF** | URL/.nc | Dimensions, variables, CF metadata | ✅ DataCube | ✅ Scientific | netcdf-to-stac |
| **CSV/TSV** | URL/.csv | Columns, geometry, dates | ✅ Table | ✅ Tabular | csv-to-stac-converter |
| **GeoJSON** | URL/.geojson | Features, geometry, properties | ✅ Feature | ✅ Spatial | auto |
| **COG/GeoTIFF** | URL/.tif | Raster, bands, georeferencing | ✅ Raster | ✅ Raster | auto |
| **OGC API** | /capabilities | Service metadata, layers | ✅ Catalog | ✅ Catalog | auto |
| **GeoPackage** | .gpkg | Tables, features, layers | ✅ Mixed | ✅ Mixed | auto |
| **Zarr** | .zarr | Dimensions, arrays, metadata | ✅ DataCube | ✅ Cloud-optimized | auto |

## Component Details

### Core Components

#### 1. **Metadata Extraction Skill** (.skills/metadata-extraction/SKILL.md)
Automatically extracts dataset properties from various sources:
- Format detection (NetCDF, CSV, GeoJSON, COG, OGC services)
- Spatial extent extraction (latitude/longitude bounds)
- Temporal range extraction (start/end dates)
- Variable/column schema parsing
- Scientific metadata collection (units, standard names, attributes)
- Web service metadata retrieval (capabilities documents)

**Outputs:**
```json
{
  "format": "netcdf|csv|geojson|...",
  "title": "extracted title",
  "description": "extracted description",
  "spatial": {"bbox": [...], "crs": "EPSG:4326"},
  "temporal": {"start": "...", "end": "..."},
  "variables": [...],
  "keywords": [...],
  "creator": "...",
  "license": "..."
}
```

#### 2. **NetCDF-to-STAC Skill** (.skills/netcdf-to-stac/SKILL.md)
Converts NetCDF scientific datasets to STAC with DataCube representation:
- Multi-dimensional array mapping to cube:variables
- CF Convention parsing (standard_name, units, dimensions)
- Spatial/temporal bounds extraction
- STAC extensions: datacube, scientific, projection, raster
- Attribute-based metadata collection

**Output Example:**
```json
{
  "stac_extensions": ["datacube", "scientific", "projection"],
  "assets": {
    "data": {
      "href": "file.nc",
      "cube:variables": {
        "temperature": {"description": "...", "unit": "K"}
      }
    }
  }
}
```

#### 3. **CSV-to-Metadata Skill** (.skills/csv-to-metadata/SKILL.md)
Analyzes CSV structure and generates STAC/DCAT metadata:
- Column type inference (string, number, date)
- Geometry detection (lat/lon, WKT columns)
- Temporal detection (date/datetime columns)
- Bounding box calculation
- Schema documentation
- Sample data extraction

**Output:** Column schema for both STAC table:columns and DCAT descriptions

### Agent Tier 1: Dispatcher

#### **Metadata Dispatcher** (.agents/.metadata-dispatcher.md)

**Entry point** for all metadata generation requests.

**Workflow:**
1. Accept URL or file path as input
2. Call metadata-extraction skill
3. Collect missing required fields from user:
   - Title, description, license, theme (DCAT)
   - Spatial/temporal extent confirmation (STAC)
4. Route to appropriate specialized agent based on:
   - Data format (NetCDF → netcdf-to-stac, CSV → csv-to-stac-converter, etc.)
   - User intent (STAC only, DCAT only, or both)
5. Invoke specialized agent with complete metadata
6. Validate and return output

**Input Examples:**
```
@metadata-dispatcher generate STAC from https://data.example.com/file.nc
@metadata-dispatcher generate DCAT from /local/data.csv
@metadata-dispatcher generate both from https://api.example.com/wfs
```

### Agent Tier 2: Generic STAC/DCAT Generators

#### **STAC Metadata Generator** (.agents/.stac-metadata-generator.md)

Generates standards-compliant STAC Items from extracted metadata:
- Extension selection based on data type
- Asset creation with proper roles
- Link generation (collection, license, derived_from)
- Semantic validation
- JSON schema compliance

**Invoked by:** Dispatcher or format-specific converters

**Outputs:** STAC Item JSON with appropriate extensions

#### **DCAT Metadata Generator** (.agents/.dcat-metadata-generator.md)

Generates DCAT Records with RDF/Linked Data output:
- Dataset/Distribution structure
- Spatial/temporal property mapping
- Theme/keyword assignment
- Conformance declaration
- RDF format support (JSON-LD, Turtle, RDF/XML)

**Invoked by:** Dispatcher or format-specific converters

**Outputs:** DCAT Record in JSON-LD, Turtle, or RDF/XML format

### Agent Tier 3: Format-Specific Converters

#### **CSV-to-STAC Converter** (.agents/.csv-to-stac-converter.md)

Specialized agent for CSV → STAC conversion:
- Table schema extraction
- Geometry column detection
- Temporal extent calculation
- STAC table extension application
- Batch conversion support

**Use when:** Converting CSV files to STAC Items with table schema

#### **CSV-to-DCAT Converter** (.agents/.csv-to-dcat-converter.md)

Specialized agent for CSV → DCAT conversion:
- Column analysis
- Keyword extraction from column names
- Theme inference
- Distribution creation
- RDF output options

**Use when:** Converting CSV to DCAT for semantic catalog integration

## Usage Workflows

### Workflow 1: NetCDF → STAC
```
User: generate STAC from https://cmems.eu/sst.nc
  ↓
Dispatcher: detect NetCDF format
  ↓
Extract: spatial bounds, time range, variables
  ↓
Prompt: title, license, platform
  ↓
STAC Generator: apply datacube + scientific extensions
  ↓
Output: STAC Item with cube:variables
```

### Workflow 2: CSV → DCAT
```
User: generate DCAT from species.csv
  ↓
Dispatcher: detect CSV format
  ↓
Extract: columns, geometry cols (lat/lon), date cols
  ↓
Prompt: title, description, theme
  ↓
DCAT Generator: map columns to keywords, create distribution
  ↓
Output: DCAT Record (JSON-LD or Turtle)
```

### Workflow 3: Batch Conversion
```
User: generate STAC catalog from /data/folder/
  ↓
Dispatcher: iterate over all files
  ↓
For each file:
  - Extract metadata
  - Route to appropriate converter
  - Generate STAC Item
  ↓
Output: catalog.json + items/*.json
```

## Extension Application Logic

### For NetCDF Data → Apply:
- **datacube** - Multi-dimensional array representation
- **scientific** - Variable descriptions, processing level, wavelength
- **projection** - CRS and coordinate reference
- **raster** - For gridded data (resolution, nodata, dtype)
- **eo** - For climate/weather data with spectral info

### For Tabular (CSV) Data → Apply:
- **table** - Column schemas and metadata
- **projection** - If geometry columns detected

### For Raster (COG/GeoTIFF) Data → Apply:
- **eo** - Spectral bands, sensor info
- **raster** - Pixel encoding, resolution
- **projection** - Geospatial reference

### For Vector (GeoJSON) Data → Apply:
- **projection** - CRS if specified
- (Default geometry handling via GeoJSON standard)

### Always Apply:
- **processing** - If data is derived or processed
- OGC Incubator extensions (osc, etc.) for scientific data

## Required Metadata Fields

### For STAC Item (minimum):
- `id` - Unique identifier (auto-generated if missing)
- `description` - Dataset description (required from user)
- `geometry` - GeoJSON geometry (auto-extracted)
- `bbox` - Bounding box [minx, miny, maxx, maxy] (auto-extracted)
- `properties.start_datetime` - ISO 8601 start date (auto-extracted or prompted)
- `properties.end_datetime` - ISO 8601 end date (auto-extracted or prompted)
- `assets` - Data links (auto-generated)
- `license` - Data license (prompted if missing)

### For DCAT Dataset (minimum):
- `dct:title` - Dataset name (prompted if missing)
- `dct:description` - Dataset description (prompted if missing)
- `dcat:distribution` - Access information (auto-generated)
- `dct:issued` - Publication date (auto-set to today)
- `dct:modified` - Last modification (auto-set to file mtime)
- `dcat:keyword` - Search keywords (auto-extracted)
- `dcat:theme` - SKOS theme URI (prompted if missing)
- `dct:license` - Data license (prompted if missing)

## Error Handling

### Common Issues & Resolution

| Error | Cause | Resolution |
|-------|-------|-----------|
| "Cannot detect format" | Unsupported file extension | Provide explicit format type |
| "Missing spatial extent" | No geometry in data | Manually specify bbox or provide context |
| "No temporal data found" | No date columns | Prompt user for time range |
| "Invalid license URI" | Non-standard license | Map to standard CC or proprietary |
| "Validation failed" | Schema non-compliance | Fix specific field (see validation report) |
